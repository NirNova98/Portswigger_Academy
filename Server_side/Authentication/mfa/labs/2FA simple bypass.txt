Vulnerability:
	After sending the correct username/password, technically the user is in a login state.
	Which allows him to pass the verification code just by surfing to a different folder in the webpage.
				
Exploit:
	Surf to a different webpage after passing the username and password.
	
Example:
	http://.../my-account
	
Prevention:
	Make the user logged in after verifying that the givin verification code is valid.
